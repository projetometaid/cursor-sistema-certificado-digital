### Agenda – Criação e Guia de Uso (09/09/2025 15:58)

Este documento registra a criação do módulo de Agenda (Agendamento) e oferece instruções objetivas para outro agente de IA e para operação do time. Mantemos foco em RPC interno, validação com Zod e arquitetura limpa.

## Visão geral
- Implementado módulo de agendamento ponta a ponta: core → infra → HTTP → wiring.
- Repositórios em JSON/in-memory para MVP e `LocalCalendarProvider` como provider de calendário local.
- Validações com Zod nos DTOs HTTP.
- Teste de fumaça para geração de disponibilidade e reserva.

## Modelo de domínio
- SchedulePolicy
  - enabled, timezone, workingDays, shifts [{ start:"HH:MM", end:"HH:MM" }]
  - slotDurationMinutes, gapBetweenSlotsMinutes
  - exceptions { holidays:[{ date, reason }], blocks:[{ startISO, endISO, reason }] }
  - maxConcurrentAppointments, bookingWindowDays, serviceDurations?
- TimeSlot: { startISO, endISO, providerId, capacity }
- Appointment: { id, providerId, customer, serviceId, startISO, endISO, status }

## Casos de uso (núcleo)
- EnableSchedulingForUserUseCase: liga/desliga agenda do usuário.
- ConfigureSchedulePolicyUseCase: persiste política de agenda.
- GenerateAvailabilityUseCase: gera slots respeitando dias, turnos, duração, exceções, capacidade e agendamentos existentes.
- ListAvailabilityUseCase: delega para geração/consulta de disponibilidade.
- BookAppointmentUseCase: valida concorrência, cria agendamento e registra no provider de calendário (local no MVP).
- BlockTimeUseCase: adiciona bloqueios manuais à política.
- SetHolidaysUseCase: define feriados.

### Endpoints
1) Habilitar/Desabilitar atendimento
  - PUT `/schedules/:providerId/enabled`
  - Body:
```json
{ "enabled": true }
```

2) Definir política
  - PUT `/schedules/:providerId/policy`
  - Body (exemplo):
```json
{
  "enabled": true,
  "timezone": "America/Sao_Paulo",
  "workingDays": [1,2,3,4,5],
  "shifts": [{ "start":"08:30", "end":"12:30" }, { "start":"13:30", "end":"17:30" }],
  "slotDurationMinutes": 10,
  "gapBetweenSlotsMinutes": 0,
  "exceptions": { "holidays": [], "blocks": [] },
  "maxConcurrentAppointments": 1,
  "bookingWindowDays": 30
}
```

3) Listar disponibilidade
  - GET `/schedules/:providerId/availability?fromISO=...&toISO=...`
  - Resposta: `{ ok, slots:[{ startISO, endISO, providerId, capacity }] }`

4) Reservar horário
  - POST `/appointments/:providerId`
  - Body (exemplo):
```json
{
  "customer": { "name": "Cliente", "email": "c@example.com" },
  "serviceId": "svc-1",
  "startISO": "2025-09-10T11:30:00.000Z",
  "endISO": "2025-09-10T11:40:00.000Z"
}
```

5) Cancelar agendamento
  - DELETE `/appointments/:id`

### Status do Projeto — 09/09/2025 - 14h40m

#### Problema
- Geração de protocolo e-CNPJ via Add/5 falhando com erro 400: "Não foi possível localizar o municipio com os parâmetros informados.".
- O token da Safeweb expirava (401) durante chamadas intermitentes (biometria/consulta prévia).

#### Causa Raiz
- Divergência de endereço entre blocos exigidos pela Safeweb: `Titular.Endereco`, `Endereco` e `ClienteNotaFiscal` precisavam conter cidade/UF e IBGE consistentes e normalizados; em especial `CodigoIbgeMunicipio`/`CodigoIbgeUF` faltando no endereço do Titular.
- Necessidade de normalização (caixa alta) e preenchimento uniforme dos campos de endereço.

#### O que foi feito
- Proxy Safeweb: implementado refresh automático do token (re-tenta com novo token ao receber 401).
- Fluxo Add/5 (e-CNPJ):
  - Primeira tentativa sem Protocolo.
  - Fallback inteligente: se falhar e for e-CNPJ, tenta `EmitirCertificadoOnline` para obter Protocolo e reenvia (mantido como robustez).
- Wrapper HTTP e-CNPJ:
  - Normalização e mapeamento ViaCEP → formato Safeweb (caixa alta, `CodigoIbgeMunicipio`, `CodigoIbgeUF`).
  - Criação de helpers reutilizáveis:
    - `mapViaCepToSafewebAddress(cep, numero)` — monta endereço nos 3 blocos.
    - `buildEcnpjPayload(...)` — monta payload completo do e-CNPJ.
    - `createPix(...)` — cria PIX com referência = protocolo.
- Limpeza de código:
  - Removido proxy Safeweb duplicado em `src/infrastructure/proxies/safeweb-proxy.js` (mantido o de `src/backend/proxies/safeweb-proxy.js`).
- Injeção de `CodigoParceiro` via env (`SAFEWEB_CODIGO_PARCEIRO=parceiro0001`).

#### Resultados
- e-CPF: OK — protocolo e PIX criados (ex.: protocolo 1008812088).
- e-CNPJ: OK — protocolos e PIX criados (ex.: 1008811787, 1008812093).

#### Como reproduzir (dev)
- Subir proxies/servidor:
  - Safeweb Proxy: porta 3003 (usa vars de ambiente abaixo).
  - API: porta 3000.
- Endpoints principais:
  - `POST /protocols/ecpf`
    - body: `{ cpf, email, telefone, cep, numero, nome, dataNascimento? }`
  - `POST /protocols/ecnpj`
    - body: `{ cnpj, cpfResponsavel, email, telefone, cepPessoa?, numeroPessoa? }`
      - Se `cepPessoa` e `numeroPessoa` forem informados, usa ViaCEP mapeado; caso contrário, usa fallback parametrizado.

#### Variáveis de ambiente relevantes
- `SAFEWEB_USERNAME`, `SAFEWEB_PASSWORD`, `SAFEWEB_CNPJ_AR`, `SAFEWEB_CODIGO_PARCEIRO=parceiro0001`
- `SAFEWEB_PROXY_PORT=3003`, `PORT=3000`
